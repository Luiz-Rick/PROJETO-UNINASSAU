🚀 Bem-vindo!

Este é um sistema completo de autenticação de usuários para a Pizzaria da Orla, desenvolvido em PHP e JavaScript.

O que o sistema oferece?

  • Cadastro de novos usuários
  • Sistema de login seguro
  • Armazenamento de dados pessoais
  • Exibição de perfil do usuário logado
  • API RESTful completa
  • Interface de teste integrada

Características

  • ✅ Sem necessidade de MySQL (usa JSON)
  • ✅ Fácil de instalar e usar
  • ✅ Segurança básica implementada
  • ✅ Documentação completa
  • ✅ Interface de teste incluída
💡 Dica: Se esta é sua primeira vez, comece pela aba "Instalação"

⚙️ Instalação e Configuração

Pré-requisitos

  • XAMPP instalado (Apache + PHP)
  • Navegador web atualizado
  • Pasta do projeto no htdocs

Passo 1: Preparar o Projeto

1 Copie a pasta do projeto para:
C:\xampp\htdocs\TEMPLATE-RestaurantePizza\

Passo 2: Iniciar o XAMPP

2 Abra o XAMPP Control Panel e:
  • Clique em "Start" ao lado de Apache
  • Espere aparecer "Running" em verde

Passo 3: Verificar Instalação

3 Abra seu navegador e acesse:
http://localhost/TEMPLATE-RestaurantePizza/

Passo 4: Testar a API

4 Clique no link de teste:
http://localhost/TEMPLATE-RestaurantePizza/teste_api.html
✅ Sucesso! Se tudo funcionou, o sistema está pronto para usar.

📱 Como Usar

Fluxo do Usuário

1 Acesse a página de Cadastro

O usuário preenche o formulário com seus dados pessoais e cria uma senha

2 Dados são salvos

Os dados são validados no JavaScript e enviados para a API PHP, que salva em JSON

3 Acesse o Login

Digite seu usuário e senha criados no cadastro

4 Veja seu Perfil

A página logado.html exibe todas as suas informações

Dados do Formulário

O sistema solicita os seguintes dados:

🔌 Endpoints da API

1. Cadastrar Usuário

POST /api_usuarios.php?acao=cadastrar
Body: { "nomeCompleto": "João Silva", "usuario": "joao.silva", "telefone": "(83) 98888-8888", "endereco": "Rua das Flores", "numero": "123", "cidade": "João Pessoa", "estado": "PB", "cep": "58045-080", "senha": "senha123" }

2. Fazer Login

POST /api_usuarios.php?acao=login
Body: { "usuario": "joao.silva", "senha": "senha123" }

3. Obter Dados do Usuário

GET /api_usuarios.php?acao=obter&id=1

4. Listar Todos os Usuários

GET /api_usuarios.php?acao=listar

5. Atualizar Perfil

PUT /api_usuarios.php?acao=atualizar
Body: { "id": 1, "telefone": "(83) 99999-9999", "cidade": "Recife" }

6. Deletar Usuário

DELETE /api_usuarios.php?acao=deletar
Body: { "id": 1 }

7. Verificar Status

GET /api_usuarios.php?acao=status
💡 Dica: Use a página teste_api.html para testar todos os endpoints visualmente!

❓ Perguntas Frequentes

P: Preciso de MySQL?

R: Não! O sistema usa JSON por padrão, que é mais simples. MySQL é opcional para versões futuras.

P: Como resetar os dados?

R: Delete o arquivo usuarios.json. Ele será recriado automaticamente na próxima requisição.

P: Posso acessar do celular?

R: Sim! Substitua "localhost" pelo IP local do computador. Por exemplo: http://192.168.1.100/...

P: Minhas senhas estão seguras?

R: As senhas são criptografadas com MD5. Para produção, recomenda-se usar bcrypt ou Argon2.

P: Posso modificar os campos do cadastro?

R: Sim! Modifique api_usuarios.php e api_client.js para adicionar/remover campos.

P: Como faço para adicionar um novo usuário manualmente?

R: Use a página teste_api.html para cadastrar usando a interface visual.

P: O que fazer se receber erro CORS?

R: Certifique-se de acessar via http://localhost/ e não abrindo os arquivos HTML diretamente.

P: Como fazer logout?

R: Clique em "Sair" na página logado.html. Seus dados serão removidos da sessão.

📞 Precisa de ajuda? Verifique o console do navegador (F12) para ver mensagens de erro detalhadas.
